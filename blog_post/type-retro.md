# 글 타입: Retro (회고)

## 용도
일정 기간의 작업, 프로젝트, 또는 특정 경험을 돌아보는 글.
"잘한 것 / 아쉬운 것 / 다르게 할 것"을 솔직하게 정리.

## 메타 섹션 (독자에게 보임)

TL;DR 다음에 배치:

```markdown
## TL;DR
- ...

> **기간**: {YYYY-MM-DD ~ YYYY-MM-DD 또는 "프로젝트명 전체"}
>
> **한 줄 요약**: {핵심 성과 또는 가장 큰 교훈}

---

## 1) 목표 vs 결과
```

## 본문 구조

```
1) 목표 vs 결과 — 처음에 뭘 하려 했고, 실제로 뭘 달성했는가
2) 잘한 것 — 유지하거나 강화할 것
3) 아쉬운 것 — 기대에 못 미친 것 (비난 아님, 사실 기반)
4) 다르게 할 것 — 구체적인 개선 방안 (추상적 "더 잘하겠다" 금지)
5) 더 생각해볼 점 — 아직 결론 못 낸 것, 장기적 관점의 열린 질문
6) 다음 액션 — 실제로 실행할 것 (체크리스트 형태 권장)
```

### 잘한 것 / 아쉬운 것 팁
- 각각 3~5개로 압축
- "XX를 했다" 같은 사실 → "그래서 YY 효과가 있었다/없었다" 까지
- 숫자로 표현 가능하면 숫자 넣기 (예: "테스트 27개 → 42개")

### 다르게 할 것 팁
- "더 열심히" 같은 추상적 표현 금지
- 구체적 행동: "다음에는 X 전에 Y를 먼저 한다"
- 실행 가능한 것만 쓴다

## 마무리 섹션: 더 생각해볼 점

"다르게 할 것" 다음, "다음 액션" 전에 배치. 회고에서 결론 내리기 어려운 열린 질문을 정리.

```markdown
## 5) 더 생각해볼 점

- **{장기 관점}**: {지금은 판단하기 이른 것 — 시간이 지나야 알 수 있는 것}
- **{구조적 질문}**: {개인 노력이 아니라 프로세스/도구/환경 차원의 개선 여지}
- **{트레이드오프}**: {개선하면 다른 게 나빠질 수 있는 양날의 검}
```

### 작성 팁
- "다르게 할 것"은 구체적 행동, "더 생각해볼 점"은 아직 답이 없는 질문
- 다음 회고에서 이 질문들을 다시 꺼내볼 수 있게 기록하는 역할
- 1~3개가 적당

## Phase 1에서 물어볼 것

1. **기간/범위**: 어떤 기간 또는 프로젝트를 회고하는 거야?
2. **핵심 성과**: 가장 뿌듯한 결과 1가지는?
3. **잘한 것**: 계속 하고 싶은 것 2~3개?
4. **아쉬운 것**: 기대에 못 미친 것 2~3개?
5. **다음에 다르게**: 구체적으로 바꿀 것?
6. **숫자/데이터**: 정량적으로 보여줄 수 있는 게 있어?
